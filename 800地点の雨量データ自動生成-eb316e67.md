>

> # 2. 地点リスト（緯度経度）を読み込む

> #### 2\. 「線形補間」の式を書く
> 
> 「この緯度経度は、地図の画像の左から何％、下から何％の位置か？」を計算し、それをBlenderの平面サイズ（例えば10m x 10m）に掛け合わせます。
> 
> **Pythonコードのロジック（概念図）：**
> 
> Python
> 
> ```
> # 地図画像の範囲（例：日本全体）
> MIN_LON = 122.0  # 左
> MAX_LON = 154.0  # 右
> MIN_LAT = 20.0   # 下
> MAX_LAT = 46.0   # 上
> 
> # Blenderの平面の大きさ（単位：メートルとか）
> PLANE_WIDTH = 10.0
> PLANE_HEIGHT = 10.0
> 
> def get_xyz(lat, lon, rain_amount):
>     # 経度(X軸)の計算： (今の経度 - 左端) / 全体の幅
>     x_ratio = (lon - MIN_LON) / (MAX_LON - MIN_LON)
>     x = x_ratio * PLANE_WIDTH
> 
>     # 緯度(Y軸)の計算： (今の緯度 - 下端) / 全体の高さ
>     y_ratio = (lat - MIN_LAT) / (MAX_LAT - MIN_LAT)
>     y = y_ratio * PLANE_HEIGHT
>     
>     # 降水量(Z軸)
>     z = rain_amount * 0.1 # スケール調整
> 
>     return (x, y, z)
> ```
> 
> この関数を使って、800個の `rain.csv` の行をループさせ、`bpy.ops.mesh.primitive_cube_add(location=(x, y, z))` すれば完成です。
> 
> * * *
> 
> ### どっちがいい？
> 
> -   **正確さ重視なら:** **BlenderGIS（A）** が圧倒的に楽です。海岸線もズレません。
>     
> -   **デザイン重視なら:** 背景を真っ黒にしたり、サイバーな見た目にするために自前の画像を貼りたい場合は **自力計算（B）** です。
>     
> 
> まずは **BlenderGIS** を試してみるのが、CS志望のエンジニアとしては「既存ライブラリの活用」という意味でスマートかもしれません。AIにコードを書かせる際も「BlenderGISで設定した座標系に合わせて、CSVから柱を立てて」と言えば通じます。

> import bpy
> 
> # ---------------------------------------------------------
> # 設定: 柱の太さと、高さの強調倍率

> # 選択しているオブジェクト（インポートした点群）を取得
> obj = bpy.context.active_object
> 
> if obj and obj.type == 'MESH':
>     # ジオメトリノードモディファイアを追加
>     mod = obj.modifiers.new(name="RainBars", type='NODES')
>     
>     # ノードツリーを新規作成
>     node_tree = bpy.data.node_groups.new(name="RainViz_GeoNodes", type='GeometryNodeTree')
>     mod.node_group = node_tree
>     
>     # --- ノードの作成関数 ---
>     def new_node(type, loc):
>         n = node_tree.nodes.new(type)
>         n.location = loc
>         return n
> 
>     # ノード配置
>     in_node = new_node('NodeGroupInput', (-200, 0))
>     out_node = new_node('NodeGroupOutput', (800, 0))
>     
>     # 1. 位置情報を取得（現在のZ位置＝雨量）
>     pos_node = new_node('GeometryNodeInputPosition', (0, 200))
>     
>     # 2. Z座標（雨量）を取り出す
>     sep_xyz = new_node('ShaderNodeSeparateXYZ', (200, 200))
>     
>     # 3. 点を地面（Z=0）に戻す
>     set_pos = new_node('GeometryNodeSetPosition', (0, 0))
>     # (Zを0にするために、PositionのXとYだけ使い、Zに0を入れたベクトルを作る処理が必要だが
>     #  簡易的に「今のZ位置」を「スケール」として使うため、位置はそのままインスタンス化して後で戻す手法をとる)
>     
>     # 簡略化ロジック:
>     # ポイントの位置にある「Z」の値こそが「雨量」である。
>     # なので、Z値を「柱の高さ(Scale Z)」として使い、
>     # 柱の配置位置は「Z=0（地表）」にする。
> 
>     # Z=0 の位置を作るノード
>     comb_xyz_flat = new_node('ShaderNodeCombineXYZ', (200, -100)) # Zはデフォルト0
>     # XとYは元の位置から取る
>     sep_xyz_raw = new_node('ShaderNodeSeparateXYZ', (0, -100))
>     
>     # 4. 柱（Cube）を生成
>     cube = new_node('GeometryNodeMeshCube', (200, 400))
>     
>     # 5. ポイントにインスタンス配置
>     instance = new_node('GeometryNodeInstanceOnPoints', (400, 0))
>     
>     # 6. スケール調整（Z高さを雨量に合わせる）
>     # 雨量(Z) * HEIGHT_SCALE
>     math_mul = new_node('ShaderNodeMath', (300, 200))
>     math_mul.operation = 'MULTIPLY'
>     math_mul.inputs[1].default_value = HEIGHT_SCALE 
>     
>     comb_scale = new_node('ShaderNodeCombineXYZ', (450, 150))
>     comb_scale.inputs[0].default_value = 1.0 # Xスケール
>     comb_scale.inputs[1].default_value = 1.0 # Yスケール
>     
>     # 7. 原点調整（柱が地面から伸びるように）
>     # Cubeの原点は中心にあるので、Z方向に半分持ち上げる必要があるが、
>     # めんどくさいのでGeonodesの "Transform Geometry" でやる
>     trans_geo = new_node('GeometryNodeTransform', (300, 500))
>     trans_geo.inputs[1].default_value = (0, 0, 0.5) # Translation Z
> 
>     # --- リンク接続 ---
>     links = node_tree.links
>     
>     # 入力メッシュ -> 位置分離 & SetPosition用
>     links.new(in_node.outputs[0], sep_xyz.inputs[0])       # Position取得用
>     links.new(in_node.outputs[0], sep_xyz_raw.inputs[0])   # X,Y座標取得用
>     
>     # X, Y座標を結合して Z=0 の座標を作る
>     links.new(sep_xyz_raw.outputs[0], comb_xyz_flat.inputs[0]) # X
>     links.new(sep_xyz_raw.outputs[1], comb_xyz_flat.inputs[1]) # Y
>     
>     # Set Position (点を地面に落とす)
>     links.new(in_node.outputs[0], set_pos.inputs[0])
>     links.new(comb_xyz_flat.outputs[0], set_pos.inputs[2]) # Position
>     
>     # ポイントにインスタンス配置
>     links.new(set_pos.outputs[0], instance.inputs[0]) # Points
>     
>     # Cube -> Transform (原点ズラし) -> Instance
>     links.new(cube.outputs[0], trans_geo.inputs[0])
>     links.new(trans_geo.outputs[0], instance.inputs[2]) # Instance
>     
>     # スケール計算 (Z位置 = 雨量)
>     links.new(sep_xyz.outputs[2], math_mul.inputs[0]) # Z -> Multiply
>     links.new(math_mul.outputs[0], comb_scale.inputs[2]) # Result -> Scale Z
>     links.new(comb_scale.outputs[0], instance.inputs[3]) # Scale
>     
>     # 出力
>     links.new(instance.outputs[0], out_node.inputs[0])
>     
>     # Cubeのサイズ調整（初期値設定）
>     cube.inputs[0].default_value = (BAR_SIZE, BAR_SIZE, 1.0) # X, Y, Z(Base)
> 
>     print("Rain Bars Generated Successfully!")
> 
> else:
>     print("エラー: インポートした点群オブジェクトを選択してください")
> ```
